# 环境变量与第三方依赖

## 1. 环境变量总表

仓库当前提供两套模板：

- `.env.example`：本地开发模板
- `.env.production.example`：生产部署模板，复制为 `.env.production`

| 变量名 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `NODE_ENV` | 是 | `production` | 运行环境 |
| `PORT` | 否 | `3000` | 应用监听端口 |
| `HOSTNAME` | 否 | `0.0.0.0` | 应用监听地址 |
| `NEXT_PUBLIC_APP_URL` | 是 | `http://localhost:3000` | 站点对外访问地址，用于 sitemap、canonical、SEO 链接 |
| `JWT_SECRET` | 是 | 无 | 管理后台登录态签名密钥 |
| `DEFAULT_ADMIN_USERNAME` | 否 | `autobes3` | 默认管理员用户名 |
| `DEFAULT_ADMIN_EMAIL` | 否 | `admin@bes3.local` | 默认管理员邮箱 |
| `DEFAULT_ADMIN_PASSWORD` | 是 | `Auto12#40Bes3` | 默认管理员密码，生产必须改掉 |
| `DATABASE_URL` | 否 | 空 | PostgreSQL 连接串，设置后优先生效 |
| `DATABASE_PATH` | 否 | `./data/bes3.db` | SQLite 文件路径 |
| `MEDIA_DRIVER` | 是 | `local` | `local` 或 `s3` |
| `MEDIA_LOCAL_ROOT` | 否 | `storage/media` | 本地媒体根目录 |
| `MEDIA_PUBLIC_BASE_URL` | 否 | 空 | S3/对象存储的对外访问前缀 |
| `S3_ENDPOINT` | 当 `MEDIA_DRIVER=s3` 时必填 | 空 | S3 / R2 / MinIO 接口地址 |
| `S3_REGION` | 当 `MEDIA_DRIVER=s3` 时建议填写 | `auto` | S3 区域 |
| `S3_BUCKET` | 当 `MEDIA_DRIVER=s3` 时必填 | 空 | Bucket 名称 |
| `S3_ACCESS_KEY_ID` | 当 `MEDIA_DRIVER=s3` 时必填 | 空 | 对象存储 Access Key |
| `S3_SECRET_ACCESS_KEY` | 当 `MEDIA_DRIVER=s3` 时必填 | 空 | 对象存储 Secret |
| `S3_FORCE_PATH_STYLE` | 否 | `false` | MinIO 常见需要设为 `true` |
| `PARTNERBOOST_AMAZON_BASE_URL` | 否 | `https://app.partnerboost.com` | PartnerBoost Amazon 接口地址 |
| `PARTNERBOOST_AMAZON_TOKEN` | Amazon 同步必填 | 空 | PartnerBoost Amazon Token |
| `PARTNERBOOST_DTC_BASE_URL` | 否 | `https://app.partnerboost.com` | PartnerBoost DTC 接口地址 |
| `PARTNERBOOST_DTC_TOKEN` | DTC 同步必填 | 空 | PartnerBoost DTC Token |
| `GEMINI_API_KEY` | AI 生成建议必填 | 空 | Gemini API Key |
| `GEMINI_MODEL` | 否 | `gemini-2.5-flash` | Gemini 模型名 |
| `PLAYWRIGHT_HEADLESS` | 否 | `true` | 是否允许 Playwright 无头兜底抓取 |
| `PINGOMATIC_ENABLED` | 否 | `false` | 是否开启 Ping-O-Matic 通知 |
| `PIPELINE_WORKER_ENABLED` | 否 | `true` | 是否启用后台流水线 worker |
| `PIPELINE_WORKER_POLL_MS` | 否 | `2500` | worker 轮询队列的时间间隔，单位毫秒 |
| `PIPELINE_WORKER_CONCURRENCY` | 否 | `1` | 单容器内允许并发执行的流水线数量，当前建议 `1-2` |

## 2. 第三方依赖

### 2.1 必选依赖

- Node.js 22+
- npm 11+
- Linux 生产环境建议 Docker 化运行

### 2.2 联盟平台

- PartnerBoost Amazon API
- PartnerBoost DTC API

如果不配置对应 Token：

- 页面仍可正常访问
- 手工导入推广链接仍可用
- 对应的联盟同步按钮会失败

### 2.3 AI 依赖

- 默认集成 Gemini

如果不配置 `GEMINI_API_KEY`：

- 系统会使用站内 fallback 文案逻辑
- 不会阻断关键词、评测、对比、SEO 生成流程
- 但内容质量和多样性会下降

### 2.4 浏览器抓取依赖

- Playwright Chromium

用途：

- 推广链接多次跳转后仍无法通过 `fetch` 到达落地页时，回退到浏览器抓取
- 对 JS 重定向页面更稳

容器部署时：

- 已提供基于 Playwright 官方镜像的 `Dockerfile`
- 不需要额外手工安装 Chromium

### 2.5 对象存储依赖

当 `MEDIA_DRIVER=s3` 时，支持：

- AWS S3
- Cloudflare R2
- MinIO
- 其他兼容 S3 协议的对象存储

## 3. 环境建议

### 3.1 本地开发

- 复制 `.env.example`
- `DATABASE_PATH=./data/bes3.db`
- `MEDIA_DRIVER=local`
- `MEDIA_LOCAL_ROOT=storage/media`
- `NEXT_PUBLIC_APP_URL=http://localhost:3000`

### 3.2 测试环境

- 推荐继续使用 SQLite + 本地卷，快速验证流程
- 若要验证生产兼容性，切到 PostgreSQL + S3

### 3.3 生产环境

建议从下面命令开始：

```bash
cp .env.production.example .env.production
npm run ops:check-env -- ./.env.production
```

如果服务器只有 Docker、没有本机 Node，也可以用容器执行同一份校验：

```bash
docker compose run --rm bes3 node /app/scripts/check-runtime-env.js
```

单机推荐：

- `DATABASE_PATH=/app/data/bes3.db`
- `MEDIA_DRIVER=local`
- `MEDIA_LOCAL_ROOT=storage/media`
- 通过宿主机卷挂载 `./data` 和 `./storage/media`

多实例推荐：

- `DATABASE_URL=postgres://...`
- `MEDIA_DRIVER=s3`
- `MEDIA_PUBLIC_BASE_URL=https://cdn.example.com/bes3-media`

## 4. 配置优先级

Bes3 当前的配置优先级为：

1. 部分关键项优先读后台设置表 `system_settings`
2. 后台无值时读取环境变量
3. 都没有时回退到代码默认值

当前已按此模式处理的重点项：

- PartnerBoost Base URL
- PartnerBoost Token
- 媒体驱动
- 媒体公开地址

## 5. 启动前校验

仓库已提供运行时配置预检脚本：

```bash
npm run ops:check-env -- ./.env.production
```

校验项包括：

- 公开站点 URL
- JWT 密钥强度
- 默认管理员密码是否仍为初始化值
- SQLite / PostgreSQL 配置是否完整
- 本地媒体或 S3 媒体配置是否完整
- PartnerBoost / Gemini 是否缺失关键凭证
- worker 是否启用，以及轮询间隔和并发数是否合法

容器启动时也会自动执行同一份预检，避免错误配置直接进入服务进程。
如果走 GHCR 手动发布，`scripts/deploy-ghcr.sh` 也会在拉起容器前先执行一次镜像内预检。

## 6. 安全要求

1. `.env.production` 不入仓库。
2. `JWT_SECRET` 必须使用随机高强度密钥。
3. 默认管理员密码只允许在首次部署期存在。
4. 对象存储密钥和联盟 Token 统一通过环境变量或密文配置中心管理。
