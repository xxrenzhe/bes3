# Bes3 优惠机会与联盟推荐汇报 Memo

## 1. 一句话结论

Bes3 已经完成从“可能会被理解成优惠券站的尝试”到“以优惠机会切入、但本质上帮助用户做购买决策的联盟导购产品”的迁移。

## 2. 为什么做这次重构

原先如果继续沿着“coupon / deals”方向走，会有三个问题：

- 用户预期容易跑偏，误以为 Bes3 提供领券、券码、券后价
- 没有稳定参考价时，折扣表达很容易失真，伤害信任和 SEO
- 页面容易堆成“便宜商品列表”，但不能真正减少购买决策负担

这次重构的核心目标不是把 Bes3 做成促销信息堆场，而是把优惠流量入口和更强的决策体验结合起来。

## 3. 现在的产品定位

当前公开定位已经收口为：

> 一个以优惠机会为切入点，但靠更清晰购买决策体验留住用户的联盟导购产品。

它对用户承诺的价值是：

- 看清这次优惠到底是不是真的
- 直接收敛到最值得优先看的少数候选
- 明确今天该买、该比、还是该等

而不是：

- 领券
- 输入券码
- 追逐虚假的“券后价”

## 4. 这次实际上上线了什么

### 4.1 路由与入口

正式优惠入口已经统一为：

- `/offers`
- `/offers/[category]`
- `/biggest-discounts`

兼容入口：

- `/deals` 只做永久跳转，不再和 `/offers` 并列竞争

同时，这些入口已经进入：

- 首页
- 目录页
- 全局页脚内链
- HTML sitemap
- Open Data / llms.txt / coverage manifest

### 4.2 联盟准入

公开商品现在必须同时满足：

- 有 slug
- 有联盟商品 ID
- 有可执行跳转目标

这意味着 Bes3 公开推荐和联盟可执行性已经统一，不再出现“能推荐但不能承接联盟转化”的公开商品。

### 4.3 优惠表达

折扣表达已经切换到更严格的规则：

- 只有在参考价可靠、来源可解释、检查时间足够新时，才公开显示 `% off`
- 如果证据不够，只显示“当前优惠机会 / 近期降价 / 接近低点”

公开层也已经从 coupon 语义切到 promotion 语义，不再把原始 coupon 文案直接暴露给用户。

### 4.4 最终推荐层

这次最关键的体验升级，是最终推荐层已经统一成：

- 同品类
- 最多 3 个候选
- 明确 1 个 winner

不足时的 fallback 也已经固定：

- 3 个及以上：标准 3 选 1
- 2 个：head-to-head
- 1 个：best available now
- 0 个：不伪造推荐，只给 wait / price watch / category updates

## 5. 业务上最重要的变化

### 5.1 增长入口更清晰

Bes3 现在可以更自然地承接：

- offers
- verified discounts
- best time to buy
- category promotions

比“coupon site”路径更稳定，也更适合长期 SEO 和品牌定位。

### 5.2 转化路径更像真实购物流程

用户不再被送到一个泛促销列表里继续犹豫，而是被带到：

- 一个更可信的优惠判断
- 一个更小的最终推荐集合
- 一个更清楚的下一步动作

这更接近高质量联盟导购应该做的事。

### 5.3 信任成本下降

这次不是靠更激进的 marketing copy 换点击，而是靠更严格的证据约束换信任。

长期价值在于：

- 降低预期落差
- 降低优惠表达失真风险
- 提高推荐解释性
- 提高用户回访与任务延续的概率

## 6. 已经验证通过的关键判断

当前已经确认成立的结论：

- 公开产品不再以“优惠券”作为核心概念
- `/offers` 已成为统一优惠入口
- `/biggest-discounts` 已成为更严格的 verified discount 专题页
- 公开商品都经过联盟可执行过滤
- 百分比优惠只在可靠参考价下显示
- offers、category、brand category、brand hub、assistant / intent、open compare 已统一到同品类最多 3 选 1

## 7. 还没有结束的部分

这轮结构和定位已经落地，但真正决定效果上限的部分，后续还要继续盯：

- reference price 覆盖率
- fresh offer 密度
- 各品类联盟可执行商品规模
- 3 选 1 在不同品类里的候选质量

也就是说，下一阶段重点不再是“要不要这么做”，而是“数据覆盖和候选质量能不能把这套结构喂饱”。

## 8. 推荐的后续汇报方式

如果这份 memo 用于产品或业务同步，建议直接围绕四个问题讲：

1. Bes3 现在是什么，而不是什么
2. 这轮上线后，用户做决策的路径具体变短了哪里
3. 这轮为什么比“虚拟优惠券”路线更稳
4. 接下来最值得持续盯的四个指标是什么

## 9. 当前判断

这轮重构已经完成“定位纠偏 + 产品结构重建 + 公开表达治理 + 决策层收口”的关键工作。

现在 Bes3 的优惠能力已经不再只是一个流量点，而是已经成为：

- 更清晰的增长入口
- 更低负担的决策层
- 更可信的省钱表达
- 更可执行的联盟转化入口
