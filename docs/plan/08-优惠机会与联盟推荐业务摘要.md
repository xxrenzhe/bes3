# Bes3 优惠机会与联盟推荐业务摘要

## 1. 当前产品是什么

Bes3 这一轮已经不再朝“优惠券站”方向发展，而是明确收口为：

> 以优惠机会切入、但本质上服务购买决策的联盟导购产品。

对外心智不是“来这里领券”，而是：

- 现在买是不是划算
- 这次优惠是不是真的
- 该买哪一个，而不是继续看更多
- 如果今天不买，下一步该怎么做

## 2. 已经完成的定位迁移

### 2.1 公开语义已经完成切换

公开产品层现在使用的是：

- offers
- promotions
- best time to buy
- verified discounts

而不是：

- coupons
- coupon codes
- voucher
- 券后价 / 领券

这意味着用户预期已经被纠正到“优惠机会 + 决策支持”，而不是“领券动作”。

### 2.2 主入口已经完成迁移

现在的正式优惠入口是：

- `/offers`
- `/offers/[category]`
- `/biggest-discounts`

`/deals` 已经只做兼容跳转，不再和主入口竞争。

## 3. 这轮上线后，用户价值发生了什么变化

### 3.1 引流价值更稳定

产品现在可以承接更广的优惠搜索需求，而不会把自己锁死在“优惠券站”这个窄语义里。

这让 Bes3 更适合覆盖：

- category offers
- best time to buy
- biggest discounts
- verified promotions

### 3.2 决策负担明显下降

这轮最重要的体验变化，不是多了一个优惠页，而是：

> 最终推荐层已经收口为“同品类最多 3 选 1”。

这直接降低了用户最后一步的认知负担。

用户看到的不是“更多便宜商品”，而是：

- 一个当前最值得优先看的 winner
- 两个以内的备选
- 为什么它赢
- 如果今天不买，下一步该做什么

### 3.3 省钱表达更可信

现在不会为了显示折扣而硬造一个“原价”。

只有在参考价可靠、来源可解释、检查时间足够新的情况下，Bes3 才会公开显示：

- `% off`
- save amount
- reference price

否则会自动降级为更安全的表述：

- 当前优惠机会
- 近期降价
- 接近低点

这对用户信任、SEO 质量和长期转化都更健康。

## 4. 已经上线的核心能力

### 4.1 优惠机会主入口

`/offers` 已经成为正式主入口，承担三件事：

- 承接全站优惠流量
- 按品类组织最终推荐层
- 把用户送回商品页、对比、shortlist 或 wait flow

### 4.2 品类优惠入口

`/offers/[category]` 已经可以：

- 承接品类优惠意图
- 在同品类内执行 3 选 1
- 用更强的品类语义支持 SEO 和内链

### 4.3 最大优惠专题页

`/biggest-discounts` 已经上线，定位是：

- 近期最大优惠机会专题
- 更严格的 verified discount 视图
- 只优先展示参考价可信的折扣机会

### 4.4 联盟准入已经收口

现在公开商品不是“有内容就能上”，而是必须满足：

- 有公开 slug
- 有联盟商品 ID
- 有实际可执行跳转路径

也就是说，公开推荐和商业可执行性已经统一了。

### 4.5 公开促销文案已经去 coupon 化

内部数据仍允许保留上游联盟字段，但公开层已经改成中性 promotion 语义，不再把原始 coupon 文案直接暴露给用户或开放接口。

## 5. 这轮最重要的业务影响

### 5.1 Bes3 的增长入口更清晰了

以前“deals / offers / coupon”这类心智容易分散。

现在入口已经收敛成一条更清晰的链路：

- 首页 / 目录 / 全局内链
- `/offers`
- `/offers/[category]`
- `/biggest-discounts`

这有利于：

- 用户认知统一
- SEO 权重集中
- 机器发现路径更一致

### 5.2 推荐结果更适合转化

以前如果用户看到的是一大堆“便宜商品”，容易继续比较、继续犹豫。

现在 Bes3 已经把关键路径改成：

- 缩小到 3 个以内
- 给 1 个 winner
- 提供 wait / price watch / category updates 作为非购买 fallback

这更符合真实购物决策，也更接近高质量联盟转化路径。

### 5.3 可信度更强

这轮不是用更激进的营销词换点击，而是用更严格的证据约束换信任。

长期看，这比“虚拟优惠券”路线更稳，因为它同时兼顾了：

- 用户预期一致性
- SEO 内容真实性
- 联盟推荐质量
- 回访和复用场景

## 6. 当前还需要持续盯的指标

这轮结构已经落地，下一阶段重点不再是“要不要这么做”，而是“做得够不够深”。

建议持续观察四类指标：

### 6.1 参考价覆盖率

重点看：

- 有多少公开 offers 能拿到可靠 reference price
- 不同品类的 verified discount 覆盖率差异

### 6.2 价格新鲜度

重点看：

- 超过 72 小时后被降级的比例
- 首页 / 榜单中 fresh offer 的稳定供给

### 6.3 联盟可执行商品规模

重点看：

- 各品类中真正满足联盟准入的商品规模
- 是否有高价值品类因为联盟覆盖不足而影响推荐质量

### 6.4 3 选 1 的候选质量

重点看：

- 不同品类下是否经常只有 1 个或 2 个有效候选
- winner 是否稳定、可解释、和用户需求匹配

## 7. 当前结论

从产品、体验、商业可执行性三个角度看，这轮重构已经完成从“优惠概念尝试”到“优惠机会型联盟导购产品”的实质迁移。

现在 Bes3 已经具备一个更清晰的公开叙事：

- 用优惠机会吸引用户
- 用更低负担的推荐结构帮助做决定
- 用联盟可执行商品承接转化
- 用 wait / shortlist / compare 把未购买用户留在同一任务里

下一阶段最值得投入的，不是再改定位，而是提升：

- reference price 覆盖率
- fresh offer 密度
- 品类深度
- winner 质量
