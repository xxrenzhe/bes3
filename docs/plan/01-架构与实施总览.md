# Bes3 架构与实施总览

## 1. 当前落地架构

- 前后端框架：`Next.js 14 + React 18 + TypeScript + Tailwind CSS`
- UI 体系：复用 `autobb` 的通用 UI 基座，按 Bes3 的品牌和后台工作台场景重做视觉层
- 数据库：默认 `SQLite`，生产可切到 `PostgreSQL`
- 鉴权模型：单管理员模型，默认管理员用户名 `autobes3`
- 媒体存储：支持 `local` 和 `s3`
- 抓取方式：先解析联盟推广链接，再抓终态商品页；默认启用 `fetch`，必要时回退到 `Playwright`
- 生成方式：优先使用 Prompt 版本管理 + Gemini；无密钥时自动回退到站内兜底文案生成
- 流水线执行：数据库持久化队列 + 单 worker 后台消费，管理后台只负责入队和查看状态

## 3. 一键工作流

Bes3 当前的核心工作流已经串成一个服务端流水线，并由后台 worker 异步消费：

1. 输入推广链接，或从 PartnerBoost 同步商品。
2. 解析推广链接，拿到最终详情页地址。
3. 抓取商品主信息、规格、价格、评分、评论摘要、商品图片、评论图片。
4. 建立或更新 `products` 与 `product_media_assets`。
5. 基于 Prompt 生成长尾关键词。
6. 生成评测文章。
7. 生成对比文章。
8. 生成 SEO Title、Meta Description、Schema。
9. 写入 `articles` 与 `seo_pages`。
10. 触发 `revalidatePath`，刷新公开站点路由。

后台对应入口：

- `Products` 页面支持手工导入推广链接。
- `Products` 页面支持 PartnerBoost Amazon / DTC 同步。
- `Products` 页面支持同步时自动将本批新商品直接入队完整流水线。
- `Products` 页面支持单个商品入队完整流水线。
- `Products` 页面支持批量入队完整流水线。
- `Products` 工作台支持将关键词、评测、对比、SEO 刷新分别入队。
- `Pipeline Runs` 页面会持续刷新队列状态与 job 时间线，并支持取消、重试。

## 4. 与 autobb 的复用关系

已优先复用或对齐的部分：

- UI 原子组件与基础交互组件
- 设置页里的 AI、代理、联盟同步配置思路
- Prompt 版本管理机制
- 联盟推广链接导入和抓取链路的总体结构
- 单次操作触发完整流水线的交互模型

明确不复用的部分：

- 多用户管理
- 与 Bes3 定位无关的广告或站群特定逻辑
- 不符合单管理员模式的后台权限设计

## 5. 生产推荐形态

### 5.1 单机稳定版

- 数据库：SQLite
- 媒体：本地卷 `storage/media`
- 部署：Docker Compose
- 适用：单台服务器、单实例运行、站点数量不大时

### 5.2 扩展版

- 数据库：PostgreSQL
- 媒体：S3 / R2 / MinIO
- 部署：容器化或多实例
- 适用：多实例、媒体量大、需要水平扩展时
