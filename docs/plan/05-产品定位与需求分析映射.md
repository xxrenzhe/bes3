# Bes3 产品定位与《需求分析》映射

## 1. 先说结论

`docs/需求分析.md` 提供的是一套可复用的**方法论**，不是 Bes3 当前要直接复刻的业务形态。

对 Bes3 真正有价值的部分只有三类：

1. **用户意图优先**：先识别用户处于“找候选 / 看深度评测 / 对比 finalists / 等价格”的哪个阶段，再决定页面入口和内容结构。
2. **结构化决策路径**：同一信息要按清晰层级组织，让用户和搜索引擎都能快速理解页面在解决什么问题。
3. **统一判断链**：不管流量从哪里来，最后都要把用户送进一条清晰的购买决策链，而不是把站点做成内容堆场。

## 2. Bes3 当前主产品是什么

Bes3 的主产品不是：

- AI 安全产品
- I-Lang 协议官网
- SEO 工具站
- 需求挖掘 SaaS

Bes3 当前主产品是：

> **一个面向消费者的导购站。**
> 它的目标是帮助用户更快完成购买决策，而不是教育用户如何做 SEO 或需求挖掘。

后台管理系统只是内部操作台，用来导入商品、抓取数据、生成页面和维护 SEO，不应该反向污染消费者站点的表达。

## 3. 从《需求分析》里提炼到 Bes3 的产品原则

### 原则 A：先服务“买家意图”，再展示“内容类型”

用户不是来找“文章类型”的，用户是来解决问题的。

因此 Bes3 公共站点的一级入口应该围绕：

- 我想先找几个候选
- 我想看一个产品值不值得买
- 我已经有两个候选，想直接比
- 我不急着买，只想等好价

### 原则 B：比较只发生在同类决策里

《需求分析》强调统一判断链。映射到 Bes3，就是：

- `compare` 不应该鼓励跨品类混比
- shortlist 需要能拆成多个 `decision paths`
- 每条路径都应回到同品类内的最终选择

### 原则 C：结构化页面比“多内容”更重要

Bes3 不应该依赖长而杂的内容堆积，而应优先保证：

- 首页负责解释 Bes3 怎么帮助用户做决定
- 搜索页负责把不同意图的人送进正确页面
- 分类页负责做主题层级入口
- review / comparison / shortlist 分别承担清晰角色

### 原则 D：转化动作保持低压力

消费者站点的主动作应该保持：

- Save to shortlist
- Add to compare
- Check current price
- Subscribe for deal or category briefing

而不是用强销售语言破坏信任。

## 4. Bes3 面向用户的能力地图

### 4.1 发现层

- 首页：解释 Bes3 的价值和使用路径
- 搜索页：按买家意图进入产品、评测、对比、指南
- 分类页 / Directory：按品类浏览

### 4.2 决策层

- Product page：事实、价格、规格、引用深度页面
- Review page：单品 verdict
- Compare page：同类产品 head-to-head
- Shortlist：用户自己的决策工作区

### 4.3 转化层

- Deals：价格窗口
- Newsletter：价格提醒 / 分类简报
- Merchant exit：低压力的 `Check price`

## 5. 当前版本的功能优先级

结合当前代码状态，Bes3 的高优先级功能应保持为：

1. **买家路径清晰化**
   - 首页与搜索页明确区分不同购买阶段
2. **shortlist 决策能力增强**
   - 同类 compare
   - mixed-category guardrails
   - decision coach
3. **内部工作流顺滑**
   - 输入商品链接
   - 抓取与规范化
   - 生成 review / comparison / SEO 页面
   - 后台可控发布

## 6. 本轮落地动作

本轮产品优化将围绕下面两件事展开：

1. 公共站点文案明确 Bes3 的消费者导购定位
2. 首页与搜索页改成按 `buyer intent` 组织入口，而不是只按页面类型或内容列表组织
