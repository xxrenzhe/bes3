### 文档 1：PRD.md  (PRD V6.0 顶级操盘手版)

# Bes3 全自动导购站群 - 产品需求文档

## 1. 业务愿景与资产定位
Bes3 是一个专注 3C 数码品类的全网底价导购测评站，定位于高利润率的自动化数字资产。
- **商业模式护城河 (二级联盟策略)**: 拒绝直接绑定 Amazon Associates，全面接入 PartnerBoost 等二级联盟网络，实现无所顾忌的合规抓取与高转化文案诱导。
- **SEO 降维打击 (老域名套利)**: 放弃新域名的信任冷启动。项目必须部署在通过拍卖收购的、带有高权重历史外链的数码/开源类过期老域名上。利用历史权重叠加 I-Lang 五层结构，直接击穿 Google 3-6 个月的沙盒期 (Sandbox)，实现发布即排名。

## 2. 核心系统边界与极简管线 (KISS 原则)
坚持“绝对不引入外部重型框架（无 OpenClaw）”，纯 Python 线性调度。
- **极简挖掘流 (`keyword_miner.py`)**: 免费抓取 Google Autocomplete 与 PAA 痛点，Gemini 过滤低意图词汇，Playwright 扫描 SERP 避开大站垄断。
- **HCU 多模态视觉防御 (`image_processor.py`)**: 本地 AI 抠图 (`rembg`) 去除官方白底，合成至预设的真实桌面背景并添加阴影，生成极具真实测评感的 WebP 格式实拍图，规避多模态 AI 降权。
- **RAG 强事实校验 (`llm_engine.py`)**: 必须采用“剥离情绪与事实”的两步生成法。Step 1 仅抽取绝对准确的硬核参数，严禁 AI 推理；Step 2 以提取的事实为基准生成主观情感评测。彻底杜绝参数幻觉导致的高退货率，死守二级联盟的 EPC (每次点击收益) 数据。

## 3. SEO 霸屏与高权重寄生引流 (Parasite Syndication)
摒弃孤立、低权重的机器社交账号，利用高权重平台实现自然秒收录：
- **Ping 节点广播**: 静态页生成并更新 `sitemap.xml` 后，自动向 `rpc.pingomatic.com` 提交广播。
- **高权重内容寄生**: 系统通过 API，自动将生成的 BLUF（结论先行）部分和 1-2 张实景图片作为“预告文章”发布至 Medium、Substack 或 LinkedIn Articles 等自带极高权重的平台。文末植入 `rel="canonical"` 原文链接，强制引导高质量蜘蛛爬取，同时截获真实的公域流量。
- **双重 Schema 霸占面积**: 强制要求生成的 JSON-LD 必须包含带有星级评分的 **`Review`** 以及提取核心痛点的 **`FAQPage`**，触发搜索结果页的富媒体展示。

## 4. 转化率 (CR) 极致优化与私域兜底
- **无压力诱导与底价叙事**: 页面顶部强化“监控全网底价”人设；全站主推按钮统一为 **"Check Current Price"** 降低防备心，并辅以动态时间戳规避客诉。
- **视觉信任系统**: 前端用醒目警告色高亮“缺点”；静态植入极客测评信任印章 `✅ Hand-tested by Alex | Ad-free independent review`。
- **退出意图漏斗 (Exit Intent Capture)**: [新增] 针对 90% 以上直接关闭网页的流失流量，前端加入极简的 Alpine.js 退出检测。当鼠标移出浏览器视口时，弹出极客风的邮件订阅模态框 ("Not ready to buy? Join 5,000+ geeks getting Alex's monthly tech deals.")，将一次性公域流量转化为可长期变现的私域资产。

## 5. 本地运维卫士 (Link Inspector)
纯本地自动化巡检机制：每周定时扫描生成的静态页外链，揪出死链或断货商品，触发告警并标记重写，确保导出去的每一滴流量都能转化为佣金。

### 文档 2：Architecture.md (系统架构与技术选型 V5.0 - 终极合规版)

# Bes3 系统架构与技术规范

## 1. 架构概览
秉持 KISS 原则与二级联盟安全策略，拒绝外部重型 Agent 框架与高危 API。系统采用“纯 Python 本地自动化流水线 + 视觉合成防御 + 社交引流收录 + 单容器手动部署”架构。

## 2. 目录结构 (Monorepo 规范)
bes3-monorepo/
├── .github/workflows/   # GitHub Actions 自动化打包配置
├── backend/             # 本地 Python 核心引擎 (生成、处理、挖掘、巡检)
│   ├── main.py          # FastAPI 服务端 (或本地异步批处理主入口)
│   ├── keyword_miner.py # 极简三步挖掘流 (Autocomplete -> Gemini 意图过滤 -> SERP 分析)
│   ├── scraper.py       # Playwright 无头抓取模块 (无视 Amazon TOS，仅抓取公开参数与白底图)
│   ├── image_processor.py # [核心防御] HCU 多模态视觉合成 (rembg 抠图 + Pillow 阴影与实景背景贴图)
│   ├── llm_engine.py    # google-genai 逻辑与 Prompt 组装 (强行植入 FAQ & Review Schema)
│   ├── builder.py       # Jinja2 渲染与 HTML 落盘模块 (同步更新 sitemap.xml)
│   ├── link_inspector.py# 本地死链与缺货巡检卫士
│   ├── ping_networks.py # [全新收录引擎] Pingomatic 广播与 Twitter/Reddit 社交外链推送
│   └── keywords_registry.json # 本地意图防撞车注册表
├── templates/           # Jinja2 HTML 模板库 (必须包含 Trust Badge、实景 WebP 占位与强引导 CTA)
├── public/              # 生成的静态产物目录 (包含生成的 .webp 实景图)
├── docs/                # 项目文档，严禁打包进镜像
├── secrets/             # 存放 .env 和社交账号 API Token，严禁打包上传
├── Dockerfile           # 生产环境单容器构建脚本 (Nginx Alpine)
└── requirements.txt     # Python 依赖 (核心: playwright, google-genai, rembg, pillow, requests)

## 3. 技术栈
- **后端**: Python 3.11+, asyncio, Playwright (负责数据与原始图片获取), rembg (AI 本地去背景), Pillow (图像处理与 WebP 压缩), google-genai, Jinja2, Pillow。。
- **前端**: HTML5, Tailwind CSS, Alpine.js。
- **运维与收录**: Docker, Nginx, GitHub Actions, Pingomatic RPC, Twitter/Reddit API。

## 4. 铁律规范
- **安全隔离**: Google API 凭证 (Service Account) 和机密文件必须呆在 secrets/ 里，通过 .gitignore 阻断泄露。
- **零外部依赖**: 线上容器只暴露 80 端口，无任何后门。自动化全靠本地 backend 脚本调度，禁止引入外部重型工作流引擎。
- **视觉去重**: 严禁将抓取到的原始白底图直接渲染到前端。所有图片必须经过 image_processor.py 处理为带阴影的实景合成图。

### 文档 3：`Workflow_Schema.md` (工作流与数据字典 V5.0 - 终极合规版)

# Bes3 工作流与高级数据结构规范

## 1. 纯 Python 异步自动化与安全收录流
1. **自动挖词 (`keyword_miner.py`)**: 
   - 截胡 Google Autocomplete 痛点词。
   - Gemini 提取高购买意图 (Buyer Intent) 词汇。
   - Playwright 轻量级扫描 SERP 避开大站锋芒，写入本地 `registry`。
2. **多模态数据准备 (`scraper.py` + `image_processor.py`)**: 
   - `asyncio` 并发抓取商品基础参数与官方图片。
   - 触发图像管线：`rembg` 本地抠出透明 PNG -> `Pillow` 生成高斯模糊底部阴影 -> 贴入预设的“Alex 极客桌”背景 -> 极限压缩并输出为 `.webp` 存入 `public/images/`。
3. **I-Lang 渲染 (`llm_engine.py` + `builder.py`)**: 
   - 组装包含底价侦探文案、FAQ Schema 和 Review Schema 的 JSON-LD。
   - Jinja2 渲染 HTML 并自动向 `public/sitemap.xml` 追加新 URL。
4. **GitOps 部署**: 代码 Push 触发 GitHub Actions 更新线上容器。
5. **白帽引流秒收录 (`ping_networks.py`)**: 
   - 部署完成后，系统立即向 `rpc.pingomatic.com` 提交 Sitemap 更新广播。
   - 同步调用 Twitter/Reddit API，发布带 Hashtag 的导购推文/帖子，利用高权重社交平台的蜘蛛将 Googlebot 引导至新页面。

## 2. 本地运维巡检流 (Link Inspector)
每周执行一次 `python backend/link_inspector.py`：
- 并发 Ping `public/` 目录下所有页面的 Affiliate 跳转终点。
- 重点检测 `404` 或页面元素包含 `Currently unavailable` 的断货情况，触发本地告警，确保导出去的流量不被浪费。

### 文档 4：`Prompt_Playbook.md` (大模型提示词工程 V4.0 - 极客版)

# Bes3 大模型 Prompt 手册

## 1. 意图过滤 Prompt (Miner Step)
**Context**: Google Autocomplete 批量抓取出的几十个候选词。
**Prompt**: 
"你是一个资深 Affiliate 营销专家。评估以下关键词的购买意图（Buyer Intent），排除纯资讯词（如 'how to clean'），保留带有明确比较、痛点或购买倾向的词（如 'alternative', 'vs', 'issue'）。对每个词输出 1-10 的购买意图评分，仅返回高于 7 分的词汇列表 JSON。"

## 2. I-Lang & GEO 结构化生成 Prompt (Generation Step)
**Context**: 抓取数据 + 目标长尾词。
**Prompt**:
"你是一名严谨的独立测评专家（化名 Alex）。针对关键词 '[Target_Keyword]' 生成符合 I-Lang 与 GEO 标准的 JSON。
极其严格的约束：
1. **Bes3 情感分析**：禁止原样摘抄评论。以独立第三方口吻提炼优缺点。
2. **富媒体 Schema 喂养**：必须基于痛点生成 3 个高质量 Q&A 放入 `faq_schema_data`；必须生成合理的 `review_schema_data` 打分。
3. **转化率优化**：`meta_description` 必须制造“好奇心缺口”，不要把结论说死。
4. **雷达打分与劝退指南**：生成 `bes3_radar_score` 和 `who_is_this_not_for`。
5. 仅输出纯 JSON。"

## 3. 前端 UI 渲染指令 (UI/UX CR 优化)
在让 AI 生成模板时，必须添加以下前端业务约束：
**"Must implement the following CR (Conversion Rate) optimizations: 
1. Call to Action (CTA) buttons must NOT say 'Buy Now'. Use 'Check Current Price on Amazon' to lower psychological resistance. 
2. Next to the main CTA, hardcode a Trust Badge: '✅ Hand-tested by Alex | Ad-free independent review'. 
3. The 'Who is this NOT for' (Cons) section must use a highly visible warning UI (e.g., red/yellow border or background) to build objective trust.
4. Implement a Mobile Sticky Bottom CTA that hides on quick scroll down, and reveals on scroll up."**

### 文档 5：DevOps.md (部署与主动收录规范 V4.0 - 极客版)

# Bes3 部署与运维规范

## 1. 核心容器与进程守护
线上保持极简，仅暴露 80 端口，无动态数据库。
*(Dockerfile 与 supervisord.conf 保持不变，只需确保 Nginx 指向最新的静态产物)*

## 2. CI/CD 流水线 (`.github/workflows/main.yml`)
- 在本地执行完批量生成并 `git push` 到 `main` 分支时触发。
- 构建 `prod-latest` 镜像并推送到 GHCR。

## 3. 手动部署 SOPT 与秒级收录 (ClawCloud)
1. 登录 ClawCloud 实例。
2. 拉取镜像: `docker pull ghcr.io/<github-username>/bes3:prod-latest`
3. 无缝替换旧容器: `docker stop bes3 && docker rm bes3` 
4. 运行新容器: `docker run -d --name bes3 -p 80:80 ghcr.io/<github-username>/bes3:prod-latest`
5. **SEO 秒级收录激活 (Indexing API)**:
   在本地或服务器执行 `python3 backend/indexing_api.py`。
   *逻辑*：脚本会读取本地的 `sitemap.xml` 或新生成的 HTML 列表，通过 Google Service Account 权限，直接向 Google Indexing API 发送 `URL_UPDATED` 请求，确保文章在 30 分钟内出现在搜索结果中。
